# Multicluster Networking

## 📘 Chapter 9 — Multicluster Networking

### 🎯 핵심 메시지

이 챕터는 **여러 쿠버네티스 클러스터를 하나의 네트워크처럼 연결하는 방법**을 설명. 특히 Cilium이 eBPF 기반으로 **클러스터 간 통신, 서비스 디스커버리, 보안, 라우팅**을 어떻게 구현하는지 집중적으로 다룸.

## 🌐 1. 왜 멀티클러스터가 필요한가

ML/AI, 금융, 글로벌 서비스 등 현대 워크로드는 다음의 요구사항이 있슴

- 지역별 클러스터 분산
- 장애 격리 및 고가용성
- 데이터 주권(데이터가 특정 국가/지역에 머물러야 하는 규정)
- 대규모 확장성
- 클러스터 간 서비스 호출 필요

멀티클러스터 네트워킹은 이런 요구를 충족시키기 위한 핵심 기술

## 🔗 2. 멀티클러스터 네트워킹 모델

Chapter 9에서는 크게 두 가지 모델을 설명

### **1) Cluster Mesh (Cilium의 대표 기능)**

- 여러 클러스터를 **평면적으로 연결**
- 각 클러스터의 Cilium 에이전트가 서로 통신
- 서비스 디스커버리 공유
- Pod 간 통신이 클러스터 경계를 넘어서도 가능
- 보안 정책(CiliumNetworkPolicy)이 클러스터 간에도 일관되게 적용

### **2) Gateway 기반 멀티클러스터**

- Gateway API를 활용한 L7 중심 멀티클러스터
- Envoy 기반 L7 라우팅
- 클러스터 간 트래픽을 HTTP/gRPC 수준에서 제어
- 서비스 메시와 유사한 구조

## 🧩 3. Cluster Mesh의 핵심 구성 요소

### **1) Cilium Cluster Mesh Daemon**

- 클러스터 간 연결을 관리
- 서비스 엔드포인트 정보를 교환

### **2) KV Store (etcd 또는 Cilium 자체 메커니즘)**

- 클러스터 간 메타데이터 공유
- 서비스 엔드포인트, IP 할당 정보 등 저장

### **3) Cross-cluster Service Discovery**

- 클러스터 A의 서비스가 클러스터 B에서도 자동으로 보임
- kube-dns/CoreDNS와 통합

## 🔐 4. 멀티클러스터 보안

멀티클러스터 환경에서는 보안이 더 중요해진다.

### Cilium이 제공하는 보안 기능

- **mTLS 기반 클러스터 간 암호화**
- **Identity 기반 정책**
    - Pod의 라벨/ID 기반으로 접근 제어
- **WireGuard/IPsec 기반 암호화**
- **정책을 클러스터 전체에 일관되게 적용**

## 🚀 5. 멀티클러스터 성능 최적화

Chapter 9에서는 성능 최적화도 다룸

- eBPF 기반 라우팅으로 hop 수 최소화
- Cross-cluster LB 최적화
- Maglev hashing으로 안정적인 분산
- 대규모 클러스터에서도 성능 저하 없이 운영

## 🧭 6. 멀티클러스터 운영 패턴

### **1) Active-Active**

- 모든 클러스터가 동시에 트래픽 처리
- 글로벌 서비스에 적합

### **2) Active-Passive**

- 장애 대비용 백업 클러스터
- 금융/규제 산업에서 많이 사용

### **3) Regional Mesh**

- 지역별 클러스터를 묶고
- 지역 간에는 L7 Gateway로 연결
- 데이터 주권 요구에 적합

## 🛠️ 7. 실제 구성 예시 (개념적)

- 클러스터 A, B, C를 Mesh로 연결
- 각 클러스터의 Cilium이 서로 엔드포인트 정보를 공유
- 서비스가 클러스터 간 자동으로 노출
- 정책은 모든 클러스터에 동일하게 적용
- 필요 시 Gateway API로 L7 트래픽만 특정 클러스터로 라우팅

## 📝 Chapter 9 핵심 요약

- 멀티클러스터는 현대 분산 시스템의 필수 요소
- Cilium은 eBPF 기반으로 고성능 멀티클러스터 네트워킹을 제공
- Cluster Mesh로 L3/L4 기반 멀티클러스터 통신
- Gateway API로 L7 기반 멀티클러스터 트래픽 제어
- 보안, 성능, 운영 패턴까지 종합적으로 다룸
